Content Collections are one of Astro's most powerful features. They give you type-safe access to your Markdown and MDX content with Zod schema validation.

## Why Content Collections?

If you've ever built a blog with a static site generator, you know the pain of loosely-typed frontmatter. Content Collections solve this by:

- **Validating** frontmatter at build time with Zod schemas
- **Typing** your content automatically in TypeScript
- **Organizing** content into named collections with separate schemas

## Setting Up

First, define your collection in `src/content.config.ts`:

```typescript
import { defineCollection } from 'astro:content';
import { glob } from 'astro/loaders';
import { z } from 'astro/zod';

const blog = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.coerce.date(),
    tags: z.array(z.string()).default([]),
    draft: z.boolean().default(false),
  }),
});

export const collections = { blog };
```

Then query it in your pages:

```typescript
const allPosts = await getCollection('blog', ({ data }) => !data.draft);
```

That's it. Type-safe, validated, and ready to render.
